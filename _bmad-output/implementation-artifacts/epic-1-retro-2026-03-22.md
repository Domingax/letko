## Challenges & Growth Areas

1. **Architecture docs behind actual installed versions** — spec said Capacitor 6 + Vite 6; actual installs were Capacitor 8 + Vite 7. Right calls technically, but each subsequent story needed explicit version notes to compensate for the drift. No single story updated `architecture.md`.

2. **Missing package: `drizzle-orm/capacitor-sqlite`** — the driver doesn't exist in drizzle-orm v0.45.1. Required building a custom migration runner using `import.meta.glob` + manual journal parsing. Non-trivial workaround that wasn't visible in the PR at the time.

3. **9 TypeScript type errors left open after story 1.4** — errors in `db/index.ts`, `file-picker.android.ts`, `file-picker.web.ts`, `filesystem.web.ts`, and secure-storage tests. These forced all CI workflows to bypass `tsc -b` via `npx vite build`. Error details:
   - `shared/db/index.ts:19` — `Property 'DB' does not exist on type 'never'` (sqlite-wasm OPFS types)
   - `file-picker.android.ts:21,35` — `'multiple' does not exist in PickFilesOptions` (@capawesome type mismatch)
   - `file-picker.web.ts:29,30,51` — `showOpenFilePicker`/`showDirectoryPicker` not in TS DOM lib
   - `filesystem.web.ts:93` — `FileSystemDirectoryHandle.entries()` missing from types
   - `secure-storage.*.test.ts:20` — `undefined` not assignable to `boolean`

4. **Workarounds not surfaced in PRs** — the custom migration runner, `npx vite build` bypass, and type error decisions were documented in story files but didn't appear in PR descriptions. Reviewers had to infer context.

5. **Schema evolved mid-story 1.2** — the `languages` lookup table and `readingProgress.sectionId` FK (replacing `sectionIndex`) were added during implementation, not planning. Sound decisions, but not reflected back into the architecture docs.

---

## Key Insights & Lessons Learned

1. **`tsc -b` is part of Done** — `tsc --noEmit` (used in `typecheck`) and `tsc -b` (used in `build`) are different. Passing `typecheck` doesn't mean the project compiles. A story is not Done if `npm run build` fails.

2. **Architecture docs must be updated in the same story that deviates** — when Vite 7 or Capacitor 8 is chosen intentionally, `architecture.md` gets updated immediately. Otherwise every subsequent story carries the cognitive overhead.

3. **Autonomous pipeline is strong — transparency layer is missing** — the dev agent knows when it's making a compromise. It needs to say so explicitly in the PR, not just in the story file. This makes the code review richer and gives the Project Lead immediate visibility.

4. **Custom workarounds deserve a name in the PR** — "I built a custom migration runner because `drizzle-orm/capacitor-sqlite` doesn't exist" is exactly the kind of thing the reviewer needs to know upfront.
